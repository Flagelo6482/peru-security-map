## 1. Descripción del proyecto

Aplicación de escritorio desarrollada con **Electron + Leaflet.js** que muestra un mapa interactivo del Perú con navegación jerárquica por niveles administrativos (Departamento → Provincia → Distrito) y puntos de incidencia clasificados por nivel de alerta (rojo/amarillo/verde). Cada punto al ser seleccionado abre un panel lateral con noticias vinculadas al lugar.

**Objetivo final:** Instalable `.exe` que funciona 100% offline en cada PC (excepto los tiles del mapa base por ahora).

---

## 2. Stack técnico definido

| Componente | Tecnología | Razón |
|---|---|---|
| App escritorio | **Electron** | Empaqueta HTML/JS como `.exe` |
| Mapa interactivo | **Leaflet.js** (CDN) | GeoJSON nativo, ligero |
| Mapa base | **CartoDB Dark** (CDN) | Sin clave API, estético |
| Datos geográficos | **GeoJSON oficial INEI/IGN** | Polígonos reales del Perú |
| Base de datos local (futuro) | **SQLite** (better-sqlite3) | Puntos de incidencia |
| Noticias (futuro) | **RSS Parser** | Medios peruanos |
| Generador instalador | **electron-builder** | Genera el `.exe` |

### Decisiones descartadas
- ❌ Three.js — innecesario para mapas 2D
- ❌ Docker — no aplica para app de escritorio local
- ❌ Servidor centralizado — la app es 100% local

## 5. Comandos para retomar el proyecto en otra PC

### Primera vez (PC local)

```bash
# 1. Clonar el repo
git clone https://github.com/Flagelo6482/peru-security-map.git
cd peru-security-map

# 2. Instalar todas las dependencias (lee package.json automáticamente)
npm install

# 3. Ejecutar la app
npm start
```

### Trabajo continuo

```bash
# Antes de empezar a trabajar
git pull

# Después de hacer cambios
git add .
git commit -m "Descripción del cambio"
git push
```

### Instalar nuevas librerías

```bash
# Librería para producción (la app la usa al correr)
npm install nombre-libreria --save

# Librería solo para desarrollo (como electron-builder)
npm install nombre-libreria --save-dev
```

## 6. Próximas tareas concretas (en orden)

### Fase 1 — Estabilizar lo que ya hay
1. Verificar que `npm start` abra la ventana sin errores
2. Confirmar que los 3 niveles (Dept → Prov → Dist) se naveguen correctamente
3. Confirmar que el breadcrumb funciona para volver a niveles anteriores
4. Si hay errores, abrir DevTools con `Ctrl+Shift+I` y revisar consola

### Fase 2 — Gestión de puntos de incidencia
1. Instalar SQLite: `npm install better-sqlite3 --save`
2. Crear `db/incidents.db` con tabla: `id, lat, lng, type, name, description, district_code`
3. Crear funciones CRUD en `renderer/db.js`
4. Renderizar puntos sobre el mapa al cargar
5. Formulario en el panel lateral para agregar puntos al hacer clic en el mapa

### Fase 3 — Noticias reales
1. Instalar RSS parser: `npm install rss-parser --save`
2. Configurar fuentes RSS:
   - El Comercio: `https://elcomercio.pe/feed/`
   - La República: `https://larepublica.pe/feed/`
   - RPP: `https://rpp.pe/feed`
   - Andina: `https://andina.pe/agencia/rss.aspx`
3. Filtrar noticias por keywords del nombre del distrito/provincia
4. Cachear noticias en SQLite para reducir consultas

### Fase 4 — Distribución
1. Instalar builder: `npm install electron-builder --save-dev`
2. Configurar `package.json` con sección `build`:
   ```json
   "build": {
     "appId": "com.peru.securitymap",
     "productName": "Mapa Seguridad Perú",
     "win": { "target": "nsis" }
   }
   ```
3. Generar instalador: `npx electron-builder --win`
4. El `.exe` queda en `dist/` listo para distribuir

### Fase 5 — Mejoras opcionales
- Sistema de búsqueda por nombre de distrito
- Heatmap de densidad de incidencias
- Exportar reportes a PDF
- Tema claro/oscuro toggle
- Tiles offline (descarga local del mapa base)
- Estadísticas con gráficos (Chart.js)

## 7. Estructura de datos esperada en SQLite (futuro)

```sql
CREATE TABLE incidents (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  lat REAL NOT NULL,
  lng REAL NOT NULL,
  type TEXT NOT NULL,           -- 'red', 'yellow', 'green'
  name TEXT NOT NULL,
  description TEXT,
  ccdd TEXT,                    -- Código departamento
  ccpp TEXT,                    -- Código provincia
  ccdi TEXT,                    -- Código distrito
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE news_cache (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source TEXT NOT NULL,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  date TEXT,
  district_name TEXT,
  fetched_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```
